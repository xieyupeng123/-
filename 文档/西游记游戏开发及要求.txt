# 一. 课程简介及考核


## 1 课程简介


以简单的图像处理和应用为载体的项目训练，对专业基础要求较低，能比较全面的训练同学们的编程能力和软件开发的工程意识，并能增强后续学习的兴趣和动力。  
Pygame是跨平台Python模块，专为电子游戏设计，包括图像、声音。建立在SDL基础上，允许实时电子游戏研发而无需被低级语言束缚， 开发者可以把精力放在游戏的架构上。这次实训利用Pygame设计完成一个简单的RPG游戏。在游戏中，玩家扮演虚拟世界中的一个或者几个特定角色在特定场景下进行游戏。本游戏的主角为孙悟空。故事为西游记中祸起观音院的故事，讲述孙悟空找到观音院的过程。

## 2 Pygame简介及安装


### 1. 简介


Pygame是跨平台Python模块，专为电子游戏设计，包括图像、声音。建立在SDL（Simple DirectMedia Layer）的基础上，允许实时电子游戏研发而无需被低级语言束缚， 开发者可以把精力放在游戏的架构上。这部分还需了解Pygame中几个重要的模块各自的作用。



### 2. 应用场景


Pygame更致力于2D游戏的开发，也就是说，你可以用Pygame写一个植物大战僵尸，但是写一个魔兽世界则相当困难……请不要做出鄙夷的目光，底层的东西永远是相通的，而且对于新手而言，从简单的2D入手才是正途。

```python
from pygame.constants import QUIT

screen = pygame.display.set_mode((800, 600), 0, 32)
background = pygame.image.load(background_image).convert()

        if event.type == QUIT:            
            exit()
    screen.blit(background, (0, 0)) 
    pygame.display.update()
```

## 3 Tiled的基本使用


### 1. 问题引入


很多游戏特别是RPG游戏中背景的制作是如何完成的，怎么样生成那些复杂游戏背景？



### 2. 简介


TiledMap是一款地图编辑器。我们常用它编辑一些相似度很大的背景,例如:超级玛丽的地图、天天酷跑、COC等。这些游戏的地图都有一个共同点那么就是有很多相同的色块组成。如果用纯代码取拼接那会很复杂，如果用TileMap去制做一个地图，那么运用起来就很简单。



### 3. 知识点


+ 图层定义 **<u>重点</u>**
+ 图块资源
+ 图块层布局
+ 对象层 **<u>重点</u>**

## 20 场景渐入渐出封装


### 1. 问题引入


场景渐入渐出功能如何设计，才能使得这个渐入渐出功能能够很好的复用？



### 2. 简介


可以采用合成复用原则，将渐入渐出功能封装成类，组合到我们村庄场景中去，这样在进入场景时候，切换场景时进行复用。并且设置场景状态根据状态的不同进行不同的alpha通道值的设置从而达到渐入渐出的效果，且方便后面第二个场景的复用。



### 3. 知识点


+ 枚举类型的状态值设定
+ 有限状态机
+ 背景渐变状态机分析如图：

<img src="https://cdn.nlark.com/yuque/0/2024/png/22579395/1716858899813-ffcbd98e-6d33-4d14-b1f3-4c90c3e7db0f.png" width="496" title="" crop="0,0,1,1" id="uad9f54fd" class="ne-image">

```python
class SceneStatus(enum.IntEnum):
    In = 1
    Normal = 2
    Out = 3
       
class FadeScene:
    def __init__(self, back_image: pygame.image):
        self.back_image = back_image
        self.alpha = 0
        self.status = SceneStatus.In

    def set_status(self, status: SceneStatus):
        self.status = status
        if status == SceneStatus.In:
            self.alpha = 0
        if status == SceneStatus.Normal:
            self.alpha = 255
        if status == SceneStatus.Out:
            self.alpha = 0

    def get_out(self):
        if self.status == SceneStatus.Out and self.alpha == 255:
            return True
        else:
            return False

    def get_back_image(self, x: int, y: int):
        if x < 0:
            x = 0
        if y < 0:
            y = 0

        if x > self.back_image.get_width() - 400:
            x = self.back_image.get_width() - 400
        if y > self.back_image.get_height() - 300:
            y = self.back_image.get_height() - 300

        temp_surface = self.back_image.subsurface((x, y, 800, 600))

        if self.status == SceneStatus.Normal:
            return temp_surface
        elif self.status == SceneStatus.In:
            temp_surface.set_alpha(self.alpha)
            black_surface = pygame.Surface((800, 600))  # black_surface.fill((0, 0, 0))
            black_surface.blit(temp_surface, (0, 0))  # 参数二为位置信息
            self.alpha += 20
            if self.alpha >= 255:
                self.alpha = 0
                self.status = SceneStatus.Normal
            return black_surface
        elif self.status == SceneStatus.Out:
            temp_surface.set_alpha(255 - self.alpha)
            black_surface = pygame.Surface((800, 600))  # black_surface.fill((0, 0, 0))
            black_surface.blit(temp_surface, (0, 0))  # 参数二为位置信息
            self.alpha += 20
            if self.alpha >= 255:
                self.alpha = 255
            return black_surface
```
